Questionário

1) Na tag ‹input› com atributo type="text", para que serve o atributo "placeholder"?

R- O atributo "placeholder" no elemento com o atributo type="text" é usado para fornecer um texto de exemplo ou uma dica para o usuário sobre o que deve ser inserido no campo de texto. Esse texto geralmente aparece em cinza claro dentro do campo de texto e desaparece quando o usuário começa a digitar. O objetivo do atributo "placeholder" é fornecer orientação aos usuários sobre o formato ou conteúdo esperado no campo de texto.

2) Na tag ‹input› com atributo type="radio" ou type="checkbox", para que serve o atributo "checked"?

R- O atributo "checked" no elemento com os atributos type="radio" ou type="checkbox" é usado para definir se o campo de seleção deve ser inicialmente marcado ou selecionado. Quando o atributo "checked" estiver incluído, o campo de seleção será exibido como selecionado ou marcado por padrão. Isso é útil quando você deseja fornecer uma opção pré-selecionada para o usuário ou quando deseja que uma opção específica seja definida por padrão.

3) Na tag ‹input› para que serve o atributo "value"?

R- atributo "valor" no elemento é usado para definir o valor associado ao campo de entrada. O valor definido no atributo "value" será enviado junto com o formulário quando o usuário enviar os dados. Além disso, o valor definido no atributo "value" também é exibido como o valor padrão no campo de entrada. Por exemplo, em um campo de texto, o atributo "valor" pode ser usado para definir um valor inicial. Em um campo de seleção (como radio ou checkbox), o atributo "value" define o valor que será enviado para o servidor quando a opção for selecionada. O atributo "value" é útil para fornecer um valor padrão ou pré-definido para o usuário, bem como para enviar e processar os dados do formulário.

4) Que atributo deve ser utilizado para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez?

R- Para agrupar vários botões de rádio em um formulário de modo que apenas um deles fique marcado de cada vez, você deve usar o atributo "name" com o mesmo valor para todos os botões de rádio que deseja agrupar. ‹ul› Por exemplo: ‹li style="square"›Opção 1‹/li› ‹li›Opção 2‹/li› ‹li›Opção 3‹/li› ‹/ul› Neste exemplo, todos os botões de rádio têm o atributo "nome" definido como "opção". Isso garante que apenas um botão de rádio possa ser selecionado de cada vez dentro do grupo. Quando o usuário seleciona um botão de rádio, os outros botões dentro do mesmo grupo são automaticamente desmarcados.

5) Para que serve a tag ‹label›? Para que serve o atributo "for" nessa tag?

R- Uma etiqueta é usado para criar um rótulo associado a um elemento de formulário, como um campo de entrada (), uma caixa de seleção. Neste exemplo, o rótulo "Nome:" está associado ao campo de texto através do atributo "for" com o valor "nome". Quando o usuário clica no rótulo, o foco é transferido para o campo de texto correspondente, facilitando a interação do usuário com o formulário. Além disso, ao clicar no rótulo, o campo de texto também é selecionado, o que é útil para melhorar a experiência do usuário em dispositivos móveis ou para pessoas com dificuldades motoras.

6) Na tag ‹option› para que serve o atributo "selected"?

R- O elemento HTML select (‹select›) representa um controle que apresenta um menu de opções. As opções dentro do menu são representadas pelo elemento ‹option›, que podem ser agrupados por elementos ‹optgroup›. As opções podem ser pré-selecionadas para o usuário.

7) Nas tags de formulário para que serve o atributo "required"?

R- Esse é um atributo booleano usado para indicar que um determinando campo de formulário é obrigatório para o envio do mesmo.

8) Para que serve a tag ‹textarea›

R- O elemento HTML ‹textarea› representa um controle de edição para uma caixa de texto, útil quando você quer permitir ao usuário informar um texto extenso em formato livre, como um comentário ou formulário de retorno.

11) Para que o valor digitado de uma senha não apareça na barra de endereços do navegador, qual deve ser o valor do atributo "method" na tag ‹form›?

O valor do atributo "method" na tag não está relacionado à visibilidade da senha na barra de endereços do navegador. O atributo "method" define o método HTTP que será usado para enviar os dados do formulário para o servidor. Para ocultar a senha na barra de endereços, você deve usar uma conexão HTTPS segura.

12) Qual valor deve ser informado no atributo "method" na tag ‹form› em um formulário que será usado para fazer upload de arquivos?

R- O valor do atributo "method" na tag para fazer upload de arquivos deve ser "POST".

13) O que as tags ‹fieldset› e ‹legend› produzem?

R- Tanto a tag ‹fieldset› quanto a tag ‹legend› podem receber os atributos id , class , title , style , dir , lang e acceskey , e os eventos onclick , ondblclick , onmousedown , onmouseup , onmouseover , onmousemove , onmouseout , onkeypress , onkeydown e onkeyup.
